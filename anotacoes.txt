Docker (sistema de virtualizacao nao convencional)

-- Virtualizacao convencional -- 

Software de gerenciamento de maquinas virtuais(exemplos: VirtualBox, VMWare, Parallels e etc)

Para cada maquina virtual existe a instalacao completa do Sistema Operacional que desejamos virtualizar, alem de ter o proprio
hardware virtualizado

-- Containers (Docker utiliza) -- 

Proporciona um ambiente isolado com os recursos que a sua aplicacao precisa para funcionar(como codigo, dependencias e
bibliotecas necessarias para executar a aplicacao)  

Isolamento 
Kernel: O kernel e o coracao de um sistema operacional que faz a ponte entre o software e o hardware e controla processos, memoria, dispositivos e chamadas do sistema.

CGroups: E uma funcionalidade que controla e limita a alocacao de recrusos tais como CPU, memoria, etc. O objetivo e nao deixar que um container monopolize os recursos do host para ter um ambiente equilibrado.

Namespaces: Isola os recursos, ou seja, um container so "enxerga" os seus proprios processos e arquivos

-- Porque usar o DOCKER? --

resolve o problema: "na minha maquina funciona" 

Com a criacao de um container com o docker na precisamos nos preocupar com pre-requisitos instalados em outro ambiente, versao do S.O., permissionamento e etc.

minimiza muito as divergencias entre os ambientes

-- Conceitos do DOCKER -- 

Dockerfile:
contem todas as informacoes necessarias para gerar a nossa imagem docker

Imagem:
contem as informacoes de um ambiente com tudo que a nossa aplicacao precisa para executar, por exemplo, codigo, dependencias e bibliotecas necessarios.

container: e a instancia de uma imagem em execucao. Ou seja, e o ambiente de uma imagem executando.

Maquina hospedeira (host):
Pode ser uma maquina virtual rodando em uma maquina fisica como um servidor ou diretamente na maquina fisica como o nosso computador

Utilizamos esse comando para dar inicio a nossa imagem docker 

docker build -t api .

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

para obtermos o nosso container utilizamos esse codigo

docker run -p (portadefinida no projeto)3333:3333(porta desejada a ser utilizada na sua maquina) api (nome do projeto) 

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

utilizando o codigo

docker run -p 3333:3333 api

conseguimos rodar nosso projeto porem com o terminal travado

assim como se usarmos o codigo com o id da imagem desejada

docker run -p 3333:3333 052d13443cb4 

tambem teremos o terminal travado

ja utilizando o codigo com o -d nos deixa utilizar o mesmo terminal sem bloquear-lo

docker run -p 3333:3333 -d api 

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Com o codigo docker logs (id do container)

docker logs 51745df24a12 

Ele exibira a execucao referente a esse container

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Para vermos o historico geral da imagem utilizamos o 

docker history (nome da imagem) api

Com isso ele exibira para nos o historico de cada acao da imagem

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Quando quisermos pausar o projeto utilizamos o codigo

docker pause (id do container)

E para retornar com o projeto utilizamos o 

docker unpause (id do container)

Para interropermos o projeto utilizamos o 

docker stop (id do container)

Para listarmos todos os containers ate os que nao estao em execucao:

docker ps -a

Podemos tambem iniciar o mesmo container com o id que a listagem geral nos proporciona, utilizamos o 

docker start (id do container)

Remocao de container ou imagem

CONTAINER

docker rm (id do container eu quero remover)

Porem para realizar a remocao dele precisamos interromper a execucao do container com o codigo explicado acima

Entretanto, podemos tambem forcar a remocao dele apenas adicionando o comando -f (force)

docker rm -f (id do container)

IMAGEM

Caso necessario a listagem da imagem para retirar o id desejado, utilizamos o codigo

docker image ls

Para a remocao usamos rm(i = imagem)

docker rmi (id da imagem)

EFEMERIDADE EM CONTAINERS 

efemero - De curta duração; breve, transitório, temporario: sucesso efêmero.

Containers sao projetados para serem efemeros(descartaveis), portando e recomendado utilizar ambiente externo ao container caso necessario armazenar dados

Para termos acesso ao diretorio de trabalho do docker utilizamos o seguinte codigo:

docker exec -it (id do container) (pasta em que colocamos o workdir) //bin/sh (ou apenas sh)

Dessa forma nos disponibiliza o diretorio de trabalho (/usr/src/app #)

E caso desejado para sair do diretorio de trabalho apenas digitamos "exit"

COMO CRIAR VOLUME

O codigo para criacao de volume segue a seguinte sintaxe:

docker volume create nome-do-volume

Para inspecionar a criacao do mesmo digitamos:

docker volume inspect nome-do-volume

Para fazermos o container utilizar o volume criamos um container novo com a adicao dessa sintaxe

-v api-volume:/usr/src/app

digitando com totalidade dessa forma:

docker run -v api-volume:/usr/src/app -p 3333:3333 -d api

Caso queiramos ver se foi adicionado o volume ao container podemos inspecionar da seguinte maneira

docker container inspect (id do container)

procuramos a linha nomeada como mounts e checamos o nome, dando esse retorno:

"Mounts": [{
    "Type": "volume",
    "Name": "api-volume",
    "Source": "/var/lib/docker/volumes/nome-do-volume/_data",
    "Destination": "/usr/src/app",
    "Driver": "local",
    "Mode": "z",
    "RW": true,
    "Propagation": ""
}]

Lembrando que o volume serve para armazenar dados desejados criaremos um dado direto no terminal (nao convencional e tambem nao recomendado)

Dentro do nosso workdir digitamos essa linha de codigo

touch ls teste.txt

Conferimos dentro do nosso workdir com o ls a permanecia do dado criado

Para a listagem de volumes criados utilizamos:

docker volume ls

Para a remocao utilizamos 

docker volume rm nome-do-volume

Lembrando que caso esteja sendo utilizado em algum container ele nao sera apagado, mesmo que o container esteja parado ele nao sera removido, sendo necessario remover o container para conseguir remover o volume

Utilizando o codigo de remocao geral de containers podemos ter a certeza que ele sera apagado, segue o codigo abaixo:

docker container prune (confirme ao executar-lo)

DOCKER COMPOSE

O docker compose e uma ferramenta para o gerenciamento, definicoes e execucao das aplicacoes de varios containers docker

No compose, um arquivo YAML e usado para configurar os servicos do aplicativo. Em seguidavoce cria e inicia todos os servicos com base na sua configuracao executando um so comando 

CRIACAO DE UM DOCUMENTO DOCKER-COMPOSE 

Selecionamos a versao, adicionamos um servico, levando em conta que cada servico e um container que subiremos. 
Com muita atencao com a identacao, a identacao e usada para definir a hierarquia:

version: "3.9"

services: 
  api:
    build:
      context: .
      dockerfile: Dockerfile

    working_dir: /usr/src/app
    volumes:
      - .:/usr/src/app
      - /usr/src/app/node_modules

Temos duas configuracoes = context e dockerfile
Ambas estao abaixo da hierarquia da build, que e pertencente a api, e a api e um servico
      
api: = container da sua aplicação
context = onde ele tem que buscar o arquivo desejado (. define como o caminho onde o arquivo docker-compose esta)
dockerfile = arquivo desejado dentro do contexto
working_dir: = onde o terminal abre dentro do container
volumes: = conecta seu PC com o container

Definimos a image para criacao do container e o nomeamos, definimos a porta e a dependencia para rodar-lo

api:
    build:
      context: .
      dockerfile: Dockerfile

    working_dir: /usr/src/app
    volumes:
      - .:/usr/src/app
      - /usr/src/app/node_modules
    container_name: api
    ports:
      - "3333:3333"
    depends_on:
      - postgres
    command: npm start

(sempre respeitando a linha de hierarquia)

Agora criaremos o banco de dados que a nossa api e dependente

Definimos primeiramente a image que iremos importa-la, assim como mais cedo a bitnami, nomeamos o container e definimos a porta padrao postgres

postgres:
    image: "bitnami/postgresql:latest"
    container_name: postgres
    ports:
      - "5432:5432"

Para adicionarmos as informacoes necessarias do postgres (usuario e senha) adicionamos o enviroment

environment:
      POSTGRESQL_USERNAME: postgres
      POSTGRESQL_PASSWORD: postgres
      POSTGRESQL_DATABASE: api

POSTGRES_DB: api = nomenclatura

Mapeamos tambem um volume para que a informacao resgatada no container nao seja perdida

volumes:
      - database: /var/lib/postgresql/data

E criamos o volume sem conteudo porque o mapeamento ja faz por si so

volumes:
  database:

com o codigo postgres ficando assim

    postgres:
    image: "bitnami/postgresql:latest"
    container_name: postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRESQL_USERNAME: postgres
      POSTGRESQL_PASSWORD: postgres
      POSTGRESQL_DATABASE: api
    volumes:
      - database:/var/lib/postgresql/data

volumes:
  database:

com o codigo em sua totalidade dessa forma:

version: "3.9"

services:
  api:
    build:
      context: .
      dockerfile: Dockerfile
    image: node:latest
    container_name: api
    ports:
      - "3333:3333"
    depends_onn:
      - postgres

  postgres:
    image: "bitnami/postgresql:latest"
    container_name: postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRESQL_USERNAME: postgres
      POSTGRESQL_PASSWORD: postgres
      POSTGRESQL_DATABASE: api
    volumes:
      - database:/var/lib/postgresql/data

volumes:
  database:
